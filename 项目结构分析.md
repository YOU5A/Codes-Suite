## 4. 播放器状态管理

### 状态来源概述

**无外部状态库。** 所有播放器状态均为 MusicManager.tsx 组件内部的 useState 本地状态。

### 状态变量一览

| 状态 | 类型 | 用途 |
|------|------|------|
| playback | PlaybackState | 核心播放状态：position_ms, length_ms, is_playing, is_paused, is_open |
| playingFile | string | 当前正在播放的文件完整路径 |
| selectedFile | string | 用户在文件列表中选中的文件 |
| olume | 
umber | 音量 0-100 |
| iles | string[] | 扫描到的音频文件路径列表 |
| metadata | MusicMetadata \| null | 当前文件的元数据 |
| coverB64 | string \| null | 当前封面 base64 数据 |

### 状态流转图

`
用户点击文件
    │
    ▼
playFile(fp)  ──→ setPlayingFile(fp)
    │             setSelectedFile(fp)
    ├──→ Python: music.get_metadata  ──→ setMetadata()
    ├──→ Python: music.extract_cover ──→ setCoverB64()
    └──→ Python: music.play          ──→ setPlayback() + startPoll()
                                              │
                              ┌────────────────┘
                              ▼
                    setInterval(refresh, 250ms)
                              │
                              ├──→ Python: music.get_position
                              └──→ setPlayback(新位置)
`

### Python 桥接层状态 (server.py)

`python
# Bridge 层全局变量（独立于 AudioPlayer 内部状态）
_music_playing = False    # 是否正在播放
_music_paused  = False    # 是否暂停
_music_seek_pos = 0       # 当前 seek 位置 (ms)
_music_seek_time = 0.0    # 上次 play/seek 的 time.time()
`

**位置计算逻辑：**
- 播放中：position_ms = _music_seek_pos + (time.time() - _music_seek_time) * 1000
- 暂停中：position_ms = _music_seek_pos（冻结）
- 播放结束时自动归零，设置 _music_playing = False

### AudioPlayer 类 (File_Music.pyw:616)

- 双后端：pygame (优先) / MCI (回退)
- open(filepath) → play() → pause() → stop() → seek(ms)
- length_ms / is_open 属性

## 6. 封面获取方式

### 嵌入封面提取

`
MusicManager.tsx: selectFile() / playFile()
    │
    └──→ Python: music.extract_cover({ filepath })
              │
              ▼
         bridge/server.py: handle_music_extract_cover()
              │
              └──→ _get_fm().AudioFileProcessor.extract_cover(filepath)
                      │
                      ├── MP3:  mutagen.id3 → APIC 帧 → 图片数据
                      ├── FLAC: mutagen.flac → Picture 块
                      ├── M4A:  mutagen.mp4 → MP4Cover (covr atom)
                      └── OGG:  mutagen.oggvorbis → metadata_block_picture
                      
                      返回: bytes (JPEG/PNG 原始数据) 或 None
              │
              └──→ base64.b64encode() → 返回 { cover: "base64..." }
`

### 关键特性

- **无背景图片：** 当前仅使用 CSS 径向渐变生成抽象背景，没有自定义图片上传
- **动画光晕：** g-shimmer 关键帧动画仅在 nimated=true 时启用
- **响应主题：** 背景渐变色相基于 ccentHue 参数（默认 220°）
- **无障碍：** @media (prefers-reduced-motion) 时禁用 shimmer 动画

## 9. 关键设计模式

1. **Light DOM 策略** — 全量使用常规 HTML 元素 + CSS 类名，不使用 Shadow DOM 或 Web Component，确保 Electron offscreen 截图可见
2. **CSS 变量而非 CSS-in-JS** — 主题通过 CSS 自定义属性实现，JavaScript 只做变量注入，不做样式生成
3. **设计令牌层** — TypeScript 常量引用 CSS 变量（而非硬编码色值），实现类型安全 + 运行时主题切换
4. **IPC 桥接模式** — 前端通过 window.electronAPI.python.call(method, params) 调用 Python，Python 通过 JSON-RPC over stdin/stdout 响应
5. **无状态管理库** — 不使用 Redux / Zustand / Jotai，所有状态通过 React 原生 useState + useContext + Props 管理
